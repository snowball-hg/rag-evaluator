# RAG 评估指标参考

## 1. Faithfulness（忠实度）/ Factuality（事实性）

**衡量内容**：生成的答案是否与检索到的上下文在事实上保持一致。答案不应包含与上下文矛盾或未在上下文中得到支持的陈述。

**典型范围**：0.0（完全不忠实）到 1.0（完全忠实）

**计算方法**：将答案分解为独立的陈述（claims），然后使用 LLM 评判器或 NLI 模型检查每个陈述是否能在检索到的上下文中找到依据。

**适用场景**：始终使用——这是最重要的 RAG 质量指标。

**局限性**：依赖于 LLM 评判器的质量；陈述必须足够原子化以便逐个验证。

## 2. Answer Relevance（答案相关性）

**衡量内容**：生成的答案是否与用户问题相关——答案在多大程度上覆盖了查询的意图。

**典型范围**：0.0（不相关）到 1.0（完全相关）

**计算方法**：从答案中反向生成合成问题，然后计算原始问题与合成问题之间的余弦相似度。

**适用场景**：检测 RAG 系统是否生成泛泛的答非所问的回答。

## 3. Context Precision（上下文精确度）

**衡量内容**：检索到的文档块中信噪比的高低。按检索排序评估每个块是否与问题相关。

**典型范围**：0.0 到 1.0

**计算方法**：按检索排名顺序评估文档块，高分意味着相关块出现在靠前位置。

**适用场景**：独立于生成环节来诊断检索质量。

## 4. Context Recall（上下文召回率）

**衡量内容**：回答问题所需的所有信息是否都出现在检索到的上下文中。

**典型范围**：0.0 到 1.0

**计算方法**：对照真实答案中的每个陈述，检查它是否能够归因于检索到的上下文。

**适用场景**：识别检索不足或缺失的文档块。

## 5. Hallucination Rate（幻觉率）

**衡量内容**：生成的回答中不能归因于检索上下文的那部分陈述所占的比例。

**典型范围**：0.0（无幻觉）到 1.0（所有陈述都是幻觉）

**计算方法**：从答案中提取陈述，逐条与上下文进行核对。无法得到支持的陈述即为幻觉。

**适用场景**：安全关键或事实性要求高的领域（医疗、法律、金融）。

## 6. 检索排序指标

| 指标 | 衡量内容 | 范围 |
|------|---------|------|
| MRR（平均倒数排名） | 第一个相关结果的排名位置 | 0.0–1.0 |
| NDCG@k（归一化折损累计增益） | 带分级相关性的排序质量 | 0.0–1.0 |
| Recall@k | 前 k 个结果中相关文档的比例 | 0.0–1.0 |
| Precision@k | 检索到的前 k 个中相关文档的比例 | 0.0–1.0 |
| MAP（平均准确率均值） | 所有查询的平均准确率 | 0.0–1.0 |

这些指标需要对每个（查询，文档）对进行人工标注的相关性判断。

## 7. 语义相似度指标

| 指标 | 方法 | 用途 |
|------|------|------|
| BERTScore | 在嵌入空间中的 token 级别余弦相似度 | 比较生成答案与参考答案 |
| BLEU | n-gram 重叠 | 短文本事实性答案 |
| ROUGE-L | 最长公共子序列 | 多句答案、摘要 |
| METEOR | 带同义词匹配的 n-gram 重叠 | 通用文本相似度 |

词汇级指标（BLEU、ROUGE）单独使用时并不适合衡量 RAG 质量——它们衡量的是表面形式，而非事实正确性。

## 8. LLM-as-Judge（LLM 评判器）配合自定义评分标准

**衡量内容**：特定领域的要求：简洁性、语气、引用准确性、完整性。

**计算方法**：定义一个带评分标准和示例的打分规则。将问题、上下文和答案提供给 LLM，要求给出分数及推理依据。

**适用场景**：当标准指标不能捕捉特定领域的质量要求时。
